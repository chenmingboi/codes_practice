这题不能用搜索，(N) 到 (10^6)，必须做线性 DP，而且只保留最后 4 位。

关键是先把转移想出来。

---

## 一、先定义状态

设 `f[n]` 表示覆盖一个 ($2\times n$) 墙壁的方法数。

如果只有 ($2\times1$) 的竖着长方形砖，那么就是普通斐波那契。
但这里还有 L 型三格砖，所以会出现“缺一角”的中间状态，因此直接只用一个 `f[n]` 不够。

再设：

* `g[n]`：覆盖 ($2\times n$) 的墙后，**右边多出来一个缺口** 的方案数
  这个缺口可以理解成最后一列只填了上面或下面一个格子，另一格没填，且由于上下对称，两种缺口数相同，所以合并成一个状态。

---

## 二、推转移

### 1. 完整状态 `f[n]`

一个完整的 ($2\times n$) 墙，最后一步有三种来源：

#### 情况 1：最后一列竖着放一个 ($2\times1$) 砖

剩下前面是 ($2\times(n-1$))

$[
f[n-1]
]$

#### 情况 2：最后两列用两个横放的 ($2\times1$) 砖铺满

剩下前面是 ($2\times(n-2)$)

$[
f[n-2]
]$

#### 情况 3：最后一部分由一个“缺口状态”加一个 L 型砖补成完整

右边缺上角或缺下角都可以，所以有两种：

$[
2g[n-1]
]$

所以：

$[
f[n]=f[n-1]+f[n-2]+2g[n-1]
]$

---

### 2. 缺口状态 `g[n]`

一个 (2\times n) 的缺口状态，最后一步也有两种来源：

#### 情况 1：从更短的缺口状态接一个横砖

$[
g[n-1]
]$

#### 情况 2：从一个完整的 ($2\times(n-2)$) 墙后面放一个 L 型砖，制造出缺口

$[
f[n-2]
]$

所以：

$[
g[n]=g[n-1]+f[n-2]
]$

---

## 三、初始条件

很容易得到：

$[
f[0]=1
]$

空墙算一种合法铺法。

$[
f[1]=1
]$

只能竖着放一个 ($2\times1$) 砖。

$[
g[0]=0,\quad g[1]=0
]$

---

## 五、为什么只要取模 10000

题目只要求最后 4 位，所以整个过程中都可以：

$[
\text{mod }10000
]$

这样不会爆。

---

## 六、代码实现

### 写法 1：数组版

```cpp
int main() {
    ios::sync_with_stdio(false);
    cin.tie(nullptr);
    int n;
    cin >> n;
    int MOD = 10000;
    vector<int>dp(n+1);
    vector<int>g(n+1);
    dp[0] = 1, dp[1] = 1;
    g[0] = 0, g[1] = 0;
    for(int i = 2;i <= n;i++) {
        dp[i] = (dp[i-1] + dp[i-2] + 2 * g[i-1]) % MOD;
        g[i] = (dp[i-2] + g[i-1]) % MOD;
    }
    cout << dp[n];
    return 0;
}
```

---

## 七、复杂度

* 时间复杂度：(O(N))
* 空间复杂度：

  * 数组版 (O(N))
  * 滚动版 (O(1))

对于 (N\le 10^6)，完全没问题。

## 八、这题最容易卡住的点

不是代码，而是**状态设计**。

你如果只想“最后放一个什么砖”，会发现 L 型砖会留下不规则边界，所以必须引入“缺口状态”
